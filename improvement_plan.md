# B站学习专注助手 - 功能改进计划

> 基于 v1.2.2，围绕行为心理学原理设计，核心目标：**降低学习启动阻力，对抗即时满足偏好**

## 问题本质

学习启动是最困难的环节——刷视频提供即时多巴胺，学习是延迟奖励。光靠意志力对抗不可靠，需要**设计机制**而非要求更强自律。

---

## 改进方案

### P0 - 自动导航到学习视频（环境设计）

**心理学原理**：环境设计（James Clear《原子习惯》）——让好习惯触手可及，坏习惯变得麻烦

**现状痛点**：分心关闭弹窗后仍停留在刷视频页面，极容易继续刷

**改进方案**：
- 检测到分心并关闭干预弹窗后，**自动跳转到白名单中的某个学习视频**
- 优先跳转到用户最近学习的视频，否则随机选一个
- 把"回到学习"的操作成本降到零
- 配合现有白名单系统，无需新增配置

**实现要点**：
```javascript
// 分心关闭弹窗后的跳转逻辑
function navigateToStudyVideo() {
    const whitelist = ConfigManager.getWhitelistArray();
    const recentBV = StatsManager.getRecentStudyBV();
    const targetBV = recentBV || whitelist[0]?.bv;
    if (targetBV) {
        window.location.href = `https://www.bilibili.com/video/${targetBV}`;
    }
}
```

**预期效果**：分心后的回归率显著提升，从"主动选择回去"变为"被动回到学习"

### P1 - 学习连续打卡系统（损失厌恶）

**心理学原理**：损失厌恶（Kahneman & Tversky）——失去已有东西的痛苦 ≈ 获得同等东西快乐的 2 倍

**现状痛点**：缺少持续学习的动力锚点，断一天不会有任何感觉

**改进方案**：
- 每天完成至少一个学习时段 → 连续天数 +1
- 浮窗/详情面板显示 🔥 连续 N 天
- 断签前给予"紧急拯救"机会：当天只要学 5 分钟也算签到
- 连续天数归零时有明显的视觉反馈（碎裂动画/灰化）
- 打卡数据持久化存储（localStorage）

**数据结构**：
```javascript
// 打卡数据
streakData: {
    currentStreak: 5,        // 当前连续天数
    longestStreak: 12,       // 最长连续天数
    lastCheckIn: '2026-04-19', // 上次签到日期
    checkInHistory: ['2026-04-15', '2026-04-16', ...] // 签到历史
}
```

**紧急拯救规则**：
- 当天未学习但连续天数 ≥ 3 时，学习满 5 分钟即可保住连续
- 连续天数 ≥ 7 时，每月有 1 次"补签"机会

### P2 - 三级渐进式阻拦（渐进摩擦）

**心理学原理**：一上来就强拦截会引发逆反心理，逐步增加摩擦力效果更好

**现状痛点**：当前干预强度单一，要么太轻被忽略，要么太强引发烦躁

**改进方案**：

| 级别 | 触发条件 | 行为 | 可关闭性 |
|------|---------|------|---------|
| 🟡 轻提醒 | 首次进入非白名单视频 | 底部 Toast：「你正在学习时段哦」+ 倒计时显示 | 自动消失 |
| 🟠 中干预 | 分心持续 3 分钟 | 弹出单词测试/承诺界面，必须交互才能关闭 | 需交互关闭 |
| 🔴 强拦截 | 分心持续 5 分钟 | 全屏遮罩 + 自动跳转学习视频（配合 P0） | 仅跳转可解除 |

**实现要点**：
- 在现有 `check()` → `showIntervention()` 流程中增加分级判断
- 用 `distractionStartTime` 计算分心持续时长
- 级别配置可在设置面板中调整阈值

---

### P3 - 两分钟启动器（微习惯）

**心理学原理**：2 分钟法则（BJ Fogg 微习惯理论）——"只做2分钟"大幅降低心理门槛，一旦开始惯性会推着你继续

**现状痛点**：想到要学 45 分钟就觉得不想开始

**改进方案**：
- 学习时段开始时，弹出微型启动界面：
  - 「只学 2 分钟？2 分钟后可以自由选择继续或退出」
- 2 分钟后弹出：「你已经学了 2 分钟，要继续吗？」
- 大多数人会选继续（沉没成本效应）
- 即使选择退出，2 分钟的学习也有价值

**UI 设计**：
```
┌─────────────────────────────┐
│  🎯 迷你启动挑战              │
│                              │
│  只学 2 分钟试试？            │
│  2 分钟后你可以自由选择        │
│                              │
│  [开始2分钟]  [跳过，直接学]   │
└─────────────────────────────┘
```

### P4 - 学习启动承诺（承诺一致性）

**心理学原理**：承诺一致性（Cialdini）——人一旦做出公开承诺，就会不自觉地保持行为一致

**现状痛点**：开始学习时没有明确目标，容易走神

**改进方案**：
- 学习时段开始前，弹出承诺界面：「接下来 45 分钟，我计划学习____」
- 写下目标后才能关闭弹窗（哪怕只写一句话）
- 学习结束时回顾：是否完成了承诺？
- 承诺记录与打卡系统联动

**数据结构**：
```javascript
commitment: {
    text: '看完高数第三章',   // 承诺内容
    startTime: 1713523200000, // 开始时间
    completed: false          // 是否完成
}
```

---

### P5 - 学习任务清单（蔡加尼克效应）

**心理学原理**：蔡加尼克效应——未完成的任务比已完成的任务在大脑中驻留更久，产生心理张力推动行动

**现状痛点**：学习缺乏具体任务感，容易漫无目的地看视频

**改进方案**：
- 在详情面板增加「今日学习任务」模块
- 用户添加 2-3 个小目标（如"看完高数第3讲""做完10道习题"）
- 未完成任务始终显示，产生心理张力
- 完成时划掉 → 释放张力 → 获得满足感
- 任务数据按天存储

## 实施路线图

### 阶段一（v1.3.0）：降低回归成本
- [ ] P0 自动导航到学习视频
- [ ] P2 三级渐进式阻拦（轻提醒 + 中干预 + 强拦截）
- 两者配合：分心 → 渐进提醒 → 最终自动跳转学习视频

### 阶段二（v1.4.0）：建立持续动力
- [ ] P1 学习连续打卡系统（含紧急拯救）
- [ ] P4 学习启动承诺
- 两者配合：承诺开场 + 打卡收尾，形成闭环

### 阶段三（v1.5.0）：降低启动门槛
- [ ] P3 两分钟启动器
- [ ] P5 学习任务清单
- 两者配合：微目标开局 + 任务清单聚焦

---

## 技术注意事项

1. **数据存储**：所有新增数据（打卡、承诺、任务）使用 localStorage，key 统一前缀 `bilibili_study_`
2. **性能**：跳转逻辑需要确保不与页面原生导航冲突
3. **渐进增强**：三级阻拦的阈值应可通过设置面板自定义
4. **暗色适配**：所有新增 UI 组件必须同步适配暗色模式，避免重蹈 v1.2.2 的问题
5. **兼容性**：确保与现有单词学习、白名单、浮窗系统无冲突
