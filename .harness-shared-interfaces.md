## 二、CSS class 命名空间

所有本插件专属 class 必须使用 `bilibili-study-` 前缀。

### 主要 class 清单

```
bilibili-study-floating-*           // 主悬浮窗（FloatingWindow）
bilibili-study-modal-*              // 模态框（ModalManager/DetailPanel）
bilibili-study-settings-*           // 设置面板
bilibili-study-multi-tab-*          // 多窗口引导
bilibili-study-telemetry-float-*    // 调试浮动窗（TelemetryUI）
bilibili-study-dark-mode            // 暗色模式切换标志
```

### 调试窗 class 完整清单（TelemetryUI 用）

| class 名 | 用途 | 定义在 |
|---------|------|-------|
| `bilibili-study-telemetry-float-window` | 浮动窗主容器 | CSS |
| `bilibili-study-telemetry-float-header` | 标题栏（可拖拽） | CSS |
| `bilibili-study-telemetry-float-title` | 标题文字 | CSS |
| `bilibili-study-telemetry-float-close` | 关闭按钮 | CSS |
| `bilibili-study-telemetry-float-tabs` | tab 容器 | CSS |
| `bilibili-study-telemetry-float-tab` | 单个 tab | CSS |
| `bilibili-study-telemetry-float-tab.active` | 选中 tab | CSS |
| `bilibili-study-telemetry-float-body` | 内容区 | CSS |
| `bilibili-study-telemetry-float-content` | 内容容器（ID） | CSS |
| `bilibili-study-telemetry-float-timeline` | Traces 时间线 | CSS |
| `bilibili-study-telemetry-float-trace-item` | 单条 trace | CSS |
| `bilibili-study-telemetry-float-trace-dot` | 时间点标记 | CSS |
| `bilibili-study-telemetry-float-trace-text` | trace 描述 | CSS |
| `bilibili-study-telemetry-float-trace-data` | trace 详情 | CSS |
| `bilibili-study-telemetry-float-remote-window` | 其他窗口卡片 | CSS |

### 暗色模式

所有 UI class 必须有对应的 `.bilibili-study-dark-mode .bilibili-study-xxx` 版本。
暗色版背景用 `#1e1e1e`，边框用 `#333`，主文用 `#e0e0e0`。

## 三、DebugTelemetry 公开 API

```javascript
DebugTelemetry.log(category, event, data)           // 记录任意事件
DebugTelemetry.logState(event, data)                 // 记录状态变更
DebugTelemetry.logMultiTab(event, data)              // 记录多窗口事件
DebugTelemetry.logIntervention(event, data)          // 记录干预事件
DebugTelemetry.logError(event, data)                 // 记录错误
DebugTelemetry.logUserAction(event, data)            // 记录用户操作
DebugTelemetry.captureSnapshot(reason)               // 捕获状态快照
DebugTelemetry.query(filterBy)                       // 按条件筛选事件
DebugTelemetry.dumpMultiWindowTrace({windowMs})      // 提取多窗口事件链
DebugTelemetry.exportJSON()                          // 导出完整日志
DebugTelemetry.getEvents()                           // 获取所有事件
DebugTelemetry.getSavedSnapshots()                   // 获取保存的快照
DebugTelemetry.clearSnapshots()                      // 清除快照

// v1.4.0: 指标
DebugTelemetry.incrementMetric(name)                 // 增加指标计数器
DebugTelemetry.getMetrics()                          // 获取所有指标值
DebugTelemetry.resetMetrics()                        // 重置指标

// 支持的指标名
metrics: guidePopupsShown, masterElections, autoNavigations,
         toastShown, errorEvents, interventionsTriggered,
         userActions, settingsChanges, whiteListChanges,
         wordsAttempted, wordsMastered
```

## 四、ConfigManager 公开 API

```javascript
ConfigManager.get()                                  // 获取全部配置
ConfigManager.save(partialConfig)                    // 保存配置（合并）
ConfigManager.getWhitelistArray()                    // 获取白名单列表
ConfigManager.isWhitelisted(bv)                      // 检查视频是否白名单
ConfigManager.addToWhitelist(bv, title)              // 添加到白名单
ConfigManager.removeFromWhitelist(index)             // 从白名单移除
ConfigManager.getDefaultReturnBV()                   // 获取默认返回视频
ConfigManager.isStudyTime()                          // 判断当前是否学习时段
ConfigManager.getInterventionProfile(level)          // 获取干预级别配置
ConfigManager.getEffectiveInterventionStages()       // 获取生效的阶段配置
ConfigManager.getVisualEffectParams(stage)           // 获取视觉效果参数
```

## 五、Eval 检查清单

每个 Eval case 必须通过以下检查：
1. grep 确认关键代码存在（如 `lastGuideResolvedAt` 存在 = 冷却期检查通过）
2. 检查模块行号范围正确
3. 检查暗色模式适配
4. 检查无 class 名不一致（JS 引用的 class 在 CSS 中有定义）

## 六、提交规范

- 版本格式：`vX.Y.Z`
- commit 格式：`"vX.Y.Z: 改动说明"`
- 每次提交前运行：`bash harness/scripts/check.sh`
- 多 Agent 并行时，每个 Agent 在自己分支上提交，由 Orchestrator 合并
