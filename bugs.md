# B站学习专注提醒助手 - Bug 记录

## 已修复的 Bug

### 1. 弹窗间隔计时逻辑错误

**问题**：`lastPopupTime` 在弹窗显示时就被设置为 `now`，而不是在弹窗**关闭后**才重置。导致如果弹窗一直显示，下次检查时 `now - lastPopupTime` 仍然小于间隔时间。

**修复**：将 `lastPopupTime = now` 移到弹窗显示之后统一设置，移除各弹窗关闭时重复设置的代码。

**代码位置**：`InterventionController.showPopupIfNeeded()`

**修复日期**：2024-04-11

---

### 2. 单词验证弹窗回答正确后延迟关闭

**问题**：回答正确后有 1 秒延迟才关闭弹窗，这期间如果 `check()` 被调用，会因为弹窗仍显示而无法触发新的弹窗。

**修复**：回答正确后立即关闭弹窗，移除延迟。

**代码位置**：`handleWordSubmit()`

**修复日期**：2024-04-11

---

### 3. 页面不可见时计时未正确暂停

**问题**：当页面不可见（如切换标签页）时，`distractionStartTime` 没有被调整，导致切换回页面时计算出错误的时间差。

**修复**：

1. 添加 `visibilitychange` 事件监听，跟踪页面隐藏时间
2. 在恢复时同时调整 `distractionStartTime` 和 `lastPopupTime`
3. 确保弹窗间隔计时不受页面隐藏影响

**代码位置**：`handleVisibilityChange()`, `check()`

**修复日期**：2024-04-11

---

### 4. 多标签页状态共享导致冲突

**问题**：虽然每个标签页有独立的 `window.__bilibiliStudyAppState`，但缺少标签页唯一标识，难以调试和追踪。

**修复**：

1. 使用 `sessionStorage` 为每个标签页生成唯一ID
2. 在状态对象中包含标签页ID
3. 在日志中输出标签页ID，便于调试

**代码位置**：`generateTabId()`, `getTabId()`, 初始化部分

**修复日期**：2024-04-11

---

### 5. 视觉干预CSS选择器优化

**问题**：需要确保播放器不受视觉干预影响，同时覆盖更多页面元素。

**修复**：

1. 为播放器选择器添加 `!important` 确保优先级
2. 添加 `.bpx-player-video-wrap` 播放器选择器
3. 添加更多页面元素选择器：`.right-container`, `.video-info-container`, `.comment-container`
4. 确保所有阶段都明确保护播放器

**代码位置**：`STYLES` 常量中的 CSS 规则

**修复日期**：2024-04-11
