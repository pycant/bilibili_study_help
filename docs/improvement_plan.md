# B站学习专注助手 - 功能改进计划

> 基于 v1.2.2，围绕行为心理学原理设计，核心目标：**降低学习启动阻力，对抗即时满足偏好**

## 问题本质

学习启动是最困难的环节——刷视频提供即时多巴胺，学习是延迟奖励。光靠意志力对抗不可靠，需要**设计机制**而非要求更强自律。

---

## 改进方案

### P0 - 自动导航到学习视频（环境设计）

**心理学原理**：环境设计（James Clear《原子习惯》）——让好习惯触手可及，坏习惯变得麻烦

**现状痛点**：分心关闭弹窗后仍停留在刷视频页面，极容易继续刷

**改进方案**：
- 检测到分心并关闭干预弹窗后，**自动跳转到白名单中的某个学习视频**
- 优先跳转到用户最近学习的视频，否则随机选一个
- 把"回到学习"的操作成本降到零
- 配合现有白名单系统，无需新增配置

**实现要点**：
```javascript
// 分心关闭弹窗后的跳转逻辑
function navigateToStudyVideo() {
    const whitelist = ConfigManager.getWhitelistArray();
    const recentBV = StatsManager.getRecentStudyBV();
    const targetBV = recentBV || whitelist[0]?.bv;
    if (targetBV) {
        window.location.href = `https://www.bilibili.com/video/${targetBV}`;
    }
}
```

**预期效果**：分心后的回归率显著提升，从"主动选择回去"变为"被动回到学习"

### P1 - 学习连续打卡系统（损失厌恶）

**心理学原理**：损失厌恶（Kahneman & Tversky）——失去已有东西的痛苦 ≈ 获得同等东西快乐的 2 倍

**现状痛点**：缺少持续学习的动力锚点，断一天不会有任何感觉

**改进方案**：
- 每天完成至少一个学习时段 → 连续天数 +1
- 浮窗/详情面板显示 🔥 连续 N 天
- 断签前给予"紧急拯救"机会：当天只要学 5 分钟也算签到
- 连续天数归零时有明显的视觉反馈（碎裂动画/灰化）
- 打卡数据持久化存储（localStorage）

**数据结构**：
```javascript
// 打卡数据
streakData: {
    currentStreak: 5,        // 当前连续天数
    longestStreak: 12,       // 最长连续天数
    lastCheckIn: '2026-04-19', // 上次签到日期
    checkInHistory: ['2026-04-15', '2026-04-16', ...] // 签到历史
}
```

**紧急拯救规则**：
- 当天未学习但连续天数 ≥ 3 时，学习满 5 分钟即可保住连续
- 连续天数 ≥ 7 时，每月有 1 次"补签"机会

### P2 - 三级渐进式阻拦（渐进摩擦）

**心理学原理**：一上来就强拦截会引发逆反心理，逐步增加摩擦力效果更好

**现状痛点**：当前干预强度单一，要么太轻被忽略，要么太强引发烦躁

**改进方案**：

| 级别 | 触发条件 | 行为 | 可关闭性 |
|------|---------|------|---------|
| 🟡 轻提醒 | 首次进入非白名单视频 | 底部 Toast：「你正在学习时段哦」+ 倒计时显示 | 自动消失 |
| 🟠 中干预 | 分心持续 3 分钟 | 弹出单词测试/承诺界面，必须交互才能关闭 | 需交互关闭 |
| 🔴 强拦截 | 分心持续 5 分钟 | 全屏遮罩 + 自动跳转学习视频（配合 P0） | 仅跳转可解除 |

**实现要点**：
- 在现有 `check()` → `showIntervention()` 流程中增加分级判断
- 用 `distractionStartTime` 计算分心持续时长
- 级别配置可在设置面板中调整阈值

---

### P3 - 两分钟启动器（微习惯）

**心理学原理**：2 分钟法则（BJ Fogg 微习惯理论）——"只做2分钟"大幅降低心理门槛，一旦开始惯性会推着你继续

**现状痛点**：想到要学 45 分钟就觉得不想开始

**改进方案**：
- 学习时段开始时，弹出微型启动界面：
  - 「只学 2 分钟？2 分钟后可以自由选择继续或退出」
- 2 分钟后弹出：「你已经学了 2 分钟，要继续吗？」
- 大多数人会选继续（沉没成本效应）
- 即使选择退出，2 分钟的学习也有价值

**UI 设计**：
```
┌─────────────────────────────┐
│  🎯 迷你启动挑战              │
│                              │
│  只学 2 分钟试试？            │
│  2 分钟后你可以自由选择        │
│                              │
│  [开始2分钟]  [跳过，直接学]   │
└─────────────────────────────┘
```

### P4 - 学习启动承诺（承诺一致性）

**心理学原理**：承诺一致性（Cialdini）——人一旦做出公开承诺，就会不自觉地保持行为一致

**现状痛点**：开始学习时没有明确目标，容易走神

**改进方案**：
- 学习时段开始前，弹出承诺界面：「接下来 45 分钟，我计划学习____」
- 写下目标后才能关闭弹窗（哪怕只写一句话）
- 学习结束时回顾：是否完成了承诺？
- 承诺记录与打卡系统联动

**数据结构**：
```javascript
commitment: {
    text: '看完高数第三章',   // 承诺内容
    startTime: 1713523200000, // 开始时间
    completed: false          // 是否完成
}
```

### P5 - 学习任务清单（蔡加尼克效应）

**心理学原理**：蔡加尼克效应——未完成的任务比已完成的任务在大脑中驻留更久，产生心理张力推动行动

**现状痛点**：学习缺乏具体任务感，容易漫无目的地看视频

**改进方案**：
- 在详情面板增加「今日学习任务」模块
- 用户添加 2-3 个小目标（如"看完高数第3讲""做完10道习题"）
- 未完成任务始终显示，产生心理张力
- 完成时划掉 → 释放张力 → 获得满足感
- 任务数据按天存储

### P6 - 奖励系统：休闲币 + 经验值等级（A+B 融合）

**心理学原理**：
- **延迟满足**：不压制"想刷视频"的欲望，而是通过学习挣取休闲时间，把欲望转化为动力
- **成长型思维**：经验值和等级提供持续进步的视觉反馈
- **规模递增**：连续学习时间越长，单位时间奖励越高，鼓励持续专注

**核心洞察——与现有弹窗机制融合**：

> 现有的分心干预弹窗（Stage 1/2）本质上就是"奖励时间不足"的信号。用户想继续浏览，但休闲币不够。当前逻辑是"你必须回答单词题才能继续浏览"，重塑为"**你的休闲币不足，答题可以赚取额外时间**"。这不是惩罚，而是挣取。

**改造思路**：
- 现在的弹窗 = 被动拦截（"你不该刷视频"）→ 用户感觉被管制
- 改造后的弹窗 = 主动挣取（"你的休闲余额不足，答题补充"）→ 用户感觉在赚东西
- 心理框架从"禁止"变为"获取"，同样的交互，完全不同的感受

**⚠️ 注意**：此功能改动范围大（涉及弹窗逻辑、状态管理、数据持久化、UI 重构），需要详细设计后再实施。以下为初步构想，后续单独讨论逻辑细节。

#### 子系统 1：休闲币（挣取休闲时间）

**赚取规则**：
| 行为 | 奖励 |
|------|------|
| 学习 3 分钟 | +1 分钟休闲币 |
| 连续学习满 30 分钟 | +10 分钟休闲币（额外奖励） |
| 单词测试正确 | +2 分钟休闲币 |
| 连续打卡 7 天 | +15 分钟休闲币 |
| 完成学习承诺 | +5 分钟休闲币 |

**使用规则**：
- 分心时弹出弹窗，显示当前休闲币余额
- 余额充足 → 可选择"使用 N 分钟休闲币"自由浏览
- 余额不足 → 弹窗提示"休闲币不足"，答题可补充
- 休闲时段结束后自动回到学习视频（配合 P0）

**约束**：
- 休闲币永不过期（过期制造焦虑，违反"奖励感"）
- 休闲币上限 120 分钟（防止囤币后放纵）
- 休闲时段结束后自动回到学习视频

**弹窗重塑示意**：
```
┌──────────────────────────────────┐
│  ⏸️ 学习时段进行中                │
│                                   │
│  已学习 23/30 分钟               │
│  ████████████████░░░░ 77%        │
│                                   │
│  💰 休闲余额：8 分钟             │
│  再坚持 7 分钟，可额外获得 10min  │
│                                   │
│  [继续学习]  [使用休闲币 8min]    │
│              [答题补充 +2min]     │
└──────────────────────────────────┘
```

#### 子系统 2：经验值与等级

**等级体系**：
| 等级 | 称号 | 所需 XP |
|------|------|---------|
| Lv.1 | 🌱 萌新学者 | 0 |
| Lv.5 | 📖 专注学徒 | 500 |
| Lv.10 | 🎓 自律达人 | 2,000 |
| Lv.20 | 🏆 学霸先锋 | 8,000 |
| Lv.50 | 👑 知识领主 | 50,000 |

**经验值来源**：
| 行为 | 经验值 |
|------|--------|
| 学习 1 分钟 | +2 XP |
| 连续学习 30 分钟 | +20 XP（额外） |
| 单词测试正确 | +5 XP |
| 连续打卡 7 天 | +100 XP |
| 完成承诺 | +30 XP |

**等级奖励**：
- 称号显示在详情面板和浮窗
- 特定等级 → 浮窗皮肤/样式变化
- 特定等级 → "免打扰通行证"（每月 1 次，跳过一次强拦截）
- 等级只升不降（降级制造挫败感，连续天数断裂已足够惩罚）

| 现有机制 | 融合后 |

| Stage 1 浮窗提醒 | 显示休闲币余额 + 继续学习进度条 |

## 实施路线图

### 阶段一（v1.3.0）：降低回归成本
- [ ] P0 自动导航到学习视频
- [ ] P2 三级渐进式阻拦（轻提醒 + 中干预 + 强拦截）
- 两者配合：分心 → 渐进提醒 → 最终自动跳转学习视频

### 阶段二（v1.4.0）：建立持续动力
- [ ] P1 学习连续打卡系统（含紧急拯救）
- [ ] P4 学习启动承诺
- 两者配合：承诺开场 + 打卡收尾，形成闭环

### 阶段三（v1.5.0）：降低启动门槛
- [ ] P3 两分钟启动器
- [ ] P5 学习任务清单
- 两者配合：微目标开局 + 任务清单聚焦

### 阶段四（v2.0.0）：奖励系统重塑 🔮
- [ ] P6 休闲币系统（挣取 + 消耗 + 余额管理）
- [ ] P6 经验值与等级系统
- [ ] P6 弹窗机制重塑：从"拦截"变为"挣取"
- 此阶段改动范围大，需单独设计详细逻辑后再实施

## 技术注意事项

1. **数据存储**：所有新增数据（打卡、承诺、任务、奖励）使用 localStorage，key 统一前缀 `bilibili_study_`
2. **性能**：跳转逻辑需要确保不与页面原生导航冲突
3. **渐进增强**：三级阻拦的阈值应可通过设置面板自定义
4. **暗色适配**：所有新增 UI 组件必须同步适配暗色模式，避免重蹈 v1.2.2 的问题
5. **兼容性**：确保与现有单词学习、白名单、浮窗系统无冲突
6. **奖励系统改动范围**：P6 涉及弹窗逻辑重构，需确保向后兼容——用户未启用奖励系统时，弹窗行为应与当前一致
7. **弹窗融合策略**：改造弹窗时保持现有交互流程不变，仅调整话术和视觉框架（从"禁止"→"挣取"），降低改坏风险
8. **休闲币经济平衡**：赚取率和消耗率需要反复调优，建议初期采用保守参数，后续根据实际数据调整
